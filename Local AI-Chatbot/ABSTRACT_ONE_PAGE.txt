ABSTRACT

Cloud-based AI chatbots raise concerns regarding data privacy, internet dependency, and recurring costs. This project presents LexaMind, a fully offline AI chatbot that operates entirely on the user's local machine while maintaining complete data sovereignty.

The fundamental limitation of standard LLMs is their static knowledge base. LexaMind addresses this through a Retrieval Augmented Generation (RAG) pipeline that enables users to upload PDF documents, from which the system extracts, chunks, and vectorizes text. When a question is asked, the system performs semantic search to retrieve the most relevant document passages and augments the LLM prompt with this context, producing accurate, document-grounded responses.

The system architecture consists of a Flask-based REST API backend with 27 endpoints and a responsive web frontend. Document processing uses PyPDF2 for text extraction with an overlapping chunking strategy (512 characters, 50-character overlap). Text chunks are vectorized using term-frequency representations and searched via cosine similarity. Chat history and document metadata are persisted in a SQLite database with three tables (conversations, messages, documents) linked by foreign key relationships. A Study Assistant module generates AI-powered summaries, quizzes, and flashcards from uploaded documents. The application is also packaged as a cross-platform desktop application using Electron.

Key features: (1) RAG-enhanced document question answering with selective document filtering, (2) multi-conversation management with persistent history, (3) AI-powered study tools, (4) complete offline functionality, and (5) model switching between multiple Ollama LLMs.

Performance: Response generation in 2-10 seconds depending on hardware. Vector search latency under 1 second. Supports multi-page PDF documents with conversation continuity across sessions.

This project demonstrates the viability of deploying context-aware conversational AI in offline environments, with applications in education, healthcare, legal review, and data-sensitive scenarios.
